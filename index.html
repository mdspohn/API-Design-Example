Introduction

The <Application Name> API is organized around REST. Our API has predictable, resource-oriented URLs, and uses HTTP response codes to indicate API errors. Many endpoints allow you to request different response formats through normal request headers, such as JSON, XML, PDF, or XSLX.

This documentation is meant to help users of the API gain a better understand of two main things - the endpoints and how to use them, and the objects returned by such requests.

Update Batch

Update tracking numbers by sending parameters in a POST request. This endpoint accepts data as JSON or XML.

Use GET to return current values in a BatchUpdate Object instead of the normal update behavior.

id int required The ID of the batch
BatchUpdate object Update information in BatchUpdate Object format
GET | POST
text/html
application/
xml
json
[base_url]/finance/batch/[id]/update
$ curl "[base_url]/finance/batch/3/update \
-H "Content-type: application/xml" \
-F file=@/path/to/xml.xml


// path/to/xml.xml file
<BatchUpdate>
<Summary id="1" trackingNumber="FOO" />
<Summary id="2" trackingNumber="BAR" />
<Payment id="3" trackingNumber="BAZ" />
<Payment id="4" trackingNumber="QUX" />
</Batchupdate>
// Status: 200
